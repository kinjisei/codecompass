**Промт, который пользователь копирует в чат другого проекта:**

```
Контекст: я делаю отдельное личное PWA-приложение CodeCompass (D:\projects\codecompass) —
глоссарий + карточки для запоминания терминов программирования. Хочу собрать туда
знания из всех своих проектов, включая этот.

Задача: проанализируй текущий проект (эту папку) — стек, архитектуру, package.json /
build-файлы, README/CLAUDE.md если есть, ключевые технические решения. Найди
технологии, инструменты, паттерны и термины, которые здесь используются и которые
полезно знать программисту-новичку. Особенно то, что специфично именно для этого
стека/проекта — базовые вещи вроде "что такое файл, коммит, API, база данных" уже
разобраны в другом месте, не повторяй их.

Результат сохрани в файл: D:\projects\_knowledge-exports\<имя-этой-папки>.md
(создай папку _knowledge-exports, если её ещё нет).

Формат — список записей, каждая такого вида:

## Название термина/технологии
- Категория: (например: Kotlin/Android, Flutter, GitLab CI/CD, Canvas/анимация —
  придумай сама по смыслу проекта)
- Коротко: одна фраза, что это такое
- Подробно: 3-5 предложений — что это, зачем нужно, как работает на базовом уровне
- Пример из этого проекта: конкретная ссылка на файл/подход в ЭТОМ проекте

Не пересказывай бизнес-логику проекта — нужны именно технические термины и понятия,
которые пригодятся при изучении программирования в целом.

В конце файла — короткая сводка (3-5 предложений): чем этот проект отличается по
стеку/подходу от обычного React+TypeScript+Supabase проекта, если отличается.
```

## Деплой (справочно, уже выполнено)

Обновление версии теперь полностью автоматическое: любой `git push` в `main` →
Vercel сам пересобирает и публикует (CI/CD). Вручную на vercel.com заходить не нужно.
Вход в GitHub с этого компьютера настроен через браузер (Git Credential Manager,
HTTPS) и сохранён — повторно логиниться не придётся.

⚠️ **Урок №2 (автодеплой из приватного репо).** Первый git-push после ручного деплоя
НЕ задеплоился: Vercel показал «Deployment Blocked — the commit author did not have
contributing access… Hobby Plan does not support collaboration for private
repositories». Причина: коммиты подписаны git-идентити `kyerbolat2004`
(`k.yerbolat.2004@gmail.com`), а владелец проекта Vercel — аккаунт `kinjisei`; на
бесплатном тарифе для ПРИВАТНОГО репо автодеплой разрешён только автору-владельцу.
**Решение:** репозиторий переведён в **public** (секретов в нём нет — всё в
localStorage) — для публичных репо это ограничение не действует, любой push
деплоится. Альтернатива (если once понадобится приватный): подписывать коммиты тем
же email, что привязан к аккаунту Vercel-владельца, либо тариф Pro.
