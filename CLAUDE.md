# CodeCompass — контекст проекта для Claude

> Этот файл читается автоматически при старте сессии Claude Code в этой папке.
> Он даёт полный контекст, чтобы продолжить работу без потери нити.

## Что это

**CodeCompass** — личное PWA-приложение-справочник для владельца проекта: он тестировщик
(QA), новичок в программировании, параллельно учится вайб-кодить (строит второй свой
проект — Recall, `D:\projects\recall-app`) и по ходу дела знакомится с Git, Supabase,
Vercel, Google AI Studio и т.д. Цель CodeCompass — собрать все эти знания в одном месте:
глоссарий с подробными объяснениями + карточки с интервальным повторением (как в Anki),
чтобы термины не забывались.

Это **не публичный продукт**, а личный инструмент одного пользователя на одном телефоне
(Samsung S25). Поэтому здесь **нет Supabase, нет аккаунтов, нет бэкенда** — все данные
(прогресс повторения) хранятся в localStorage прямо в браузере/PWA.

## Язык общения

Отвечай пользователю **по-русски**, простыми словами, без непояснённого жаргона —
он новичок в программировании. Если используешь термин, которого может не быть в
его словаре, поясни его в паре слов или сошлись на соответствующую карточку в
глоссарии (`src/content/*.ts`).

## Стек

- **Vite + React 19 + TypeScript + Tailwind v4** (`@tailwindcss/vite`), PWA (`vite-plugin-pwa`)
- **ts-fsrs** — тот же алгоритм интервального повторения, что и в Recall
- Хранилище прогресса — **localStorage** (без бэкенда, см. `src/lib/srs.ts`)
- Хостинг (когда дойдём до публикации) — **Vercel**, так же как Recall

Стек и визуальный стиль (Tailwind-классы, компоненты Button/Card, нижняя навигация)
намеренно скопированы с Recall — чтобы пользователю было легко переключаться между
двумя своими проектами и не путаться в двух разных подходах.

## Структура

```
src/
  types/       index.ts — Category, GlossaryEntry, Rating, ReviewState + Course/Lesson/QuizQuestion
  content/     categories.ts, vibecoding.ts, git.ts, webdev.ts, index.ts (allEntries)
               courses/ — курсы: <курс>/<модуль>.ts (уроки с квизами), index.ts (allCourses)
  lib/         srs.ts — FSRS поверх localStorage; courseProgress.ts — прогресс уроков;
               backup.ts — экспорт/импорт всего прогресса в JSON
  components/  Button, Card, Layout, BottomNav
  features/    home/HomePage, courses/ (CoursesPage, CoursePage, LessonPage),
               glossary/GlossaryPage, cards/CardsPage, settings/SettingsPage
```

Нижняя навигация: Главная / Курсы / Глоссарий / Карточки. Настройки — шестерёнка на главной.

## Модель контента

Каждый термин — объект `GlossaryEntry` в `src/content/<тема>.ts`:

```ts
{
  id: 'git-commit',        // уникальный id, префикс = категория
  categoryId: 'git',
  term: 'Коммит (commit)',
  short: '...',            // короткая формулировка — вопрос на карточке
  detail: '...',           // подробное объяснение — глоссарий и обратная сторона карточки
  example: '...',          // пример из практики пользователя (Recall/Supabase/Vercel)
  related: ['git-repo'],   // id связанных терминов (пока не используется в UI, задел на будущее)
}
```

Добавить новую тему = создать `src/content/<тема>.ts` с массивом `GlossaryEntry[]`,
добавить категорию в `src/content/categories.ts` и подключить массив в `src/content/index.ts`.

## Как запустить

```
npm install      # если нужно
npm run dev      # http://localhost:5173
npm run build    # проверка типов (tsc -b) + сборка
```

## Статус и план

Актуальный список тем — готовых и будущих — веду в `ROADMAP.md`. Начинай сессию с
чтения этого файла, если нужно понять «на чём остановились» и что делать дальше —
пользователю не нужно самому формулировать промпты, достаточно написать «продолжай».

## Правила

- Простой русский язык, без канцелярита и непояснённого жаргона.
- Каждая новая тема — отдельный файл в `src/content/`, по образцу существующих.
- Примеры в карточках `example` по возможности привязывать к реальной практике
  пользователя (его же проект Recall, Supabase, Vercel, Claude Code) — так лучше
  запоминается.
- После изменений проверять `npm run build`.
- Ничего не коммитить без явной просьбы пользователя (см. общие правила Claude Code).
