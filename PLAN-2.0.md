# План CodeCompass 2.0 — из справочника в учебное приложение

> Этот файл — детальный план большой доработки. Читать вместе с `ROADMAP.md`
> (там — история контента глоссария). Когда фаза выполнена — отмечать здесь ✅.
> Статус: **в работе — Фазы 0 и 1 готовы** (июль 2026).

## 1. Цель

Сейчас CodeCompass — глоссарий + карточки. Цель — сделать из него **приложение для
обучения**: структурированные курсы с уроками, проверкой знаний (квизы) и связкой
с уже существующими карточками повторения. Плюс собрать его как **APK** для Samsung S25.

Три новых курса (по запросу владельца):

1. **Промпт-инжиниринг** — как получать от AI то, что нужно.
2. **Нейросети и автоматизация** — как устроен AI и как заставить его работать за тебя.
3. **Эмоциональный интеллект** — самосознание, саморегуляция, эмпатия, коммуникация.

**Главный приоритет — качество информации.** Лучше меньше уроков, но каждый —
точный, проверенный, с примерами из реальной жизни пользователя (QA-работа,
Recall, CodeCompass, Claude Code).

## 2. Что уже есть (аудит, июль 2026)

- Движок: Vite + React 19 + TS + Tailwind v4, PWA, ts-fsrs поверх localStorage.
- Контент: 229 терминов / 27 тем, всё в `src/content/*.ts`.
- Страницы: Главная (прогресс по темам), Глоссарий (поиск + фильтры + related-чипы),
  Карточки (FSRS-очередь, лимит 30).
- Деплой: https://codecompass-ruddy.vercel.app, автодеплой по push в `main`.
- Слабые места: прогресс живёт только в localStorage (нет резервной копии);
  нет понятия «урок» или «курс»; поиск не смотрит в `detail`; нет мотивационных
  механик (стрик).

## 3. Архитектура курсов

### 3.1 Модель данных (новые типы в `src/types/index.ts`)

```ts
interface Course {
  id: string            // 'prompting' | 'ai-auto' | 'eq'
  title: string
  icon: string
  description: string   // зачем проходить, что получишь
  modules: CourseModule[]
}

interface CourseModule {
  id: string
  title: string
  intro: string         // 1-2 фразы, о чём модуль
  lessons: Lesson[]
}

interface Lesson {
  id: string            // префикс курса: 'pr-1-1'
  title: string
  minutes: number       // честная оценка времени чтения (3-7 мин)
  blocks: LessonBlock[] // содержимое урока
  quiz: QuizQuestion[]  // 3-5 вопросов в конце
  termIds?: string[]    // термины глоссария, введённые в уроке → уходят в карточки
}

type LessonBlock =
  | { type: 'p'; text: string }                    // абзац
  | { type: 'h'; text: string }                    // подзаголовок
  | { type: 'example'; title?: string; text: string } // разобранный пример
  | { type: 'tip'; text: string }                  // 💡 совет
  | { type: 'warn'; text: string }                 // ⚠️ частая ошибка
  | { type: 'try'; text: string }                  // 🧪 «попробуй сам» — практика
  | { type: 'code'; text: string }                 // промпт/код моноширинно

interface QuizQuestion {
  id: string
  question: string
  options: string[]      // 3-4 варианта
  correct: number        // индекс правильного
  explain: string        // почему так — показывается ПОСЛЕ ответа, и при верном тоже
}
```

Принципы: контент — обычные TS-файлы как сейчас (никакого бэкенда); один модуль =
один файл `src/content/courses/<курс>/<модуль>.ts`; курс собирается в
`src/content/courses/<курс>/index.ts`.

### 3.2 Прогресс курсов (`src/lib/courseProgress.ts`)

localStorage-ключ `codecompass.course_progress.v1`:

```ts
Record<lessonId, { completedAt: string; quizBest: number; quizTotal: number }>
```

Функции: `getLessonProgress`, `completeLesson`, `getCourseProgress` (процент по курсу
и по модулю — для главной и списков).

### 3.3 UI

- Нижняя навигация: **Главная / Курсы / Глоссарий / Карточки** (4 вкладки).
- `/courses` — список курсов с прогрессом.
- `/courses/:courseId` — модули и уроки, галочки пройденного, кнопка «Продолжить»
  (первый непройденный урок).
- `/courses/:courseId/:lessonId` — экран урока: блоки → квиз → «Урок пройден».
  Внизу — «Следующий урок». Термины урока — чипами со ссылкой в глоссарий.
- Квиз: по одному вопросу, выбор варианта → подсветка правильного + объяснение →
  дальше. В конце — счёт; пересдача не ограничена, сохраняется лучший результат.
- Главная: сверху блок «Продолжить обучение» (текущий урок), ниже — как сейчас.
- Термины из `termIds` автоматически попадают в общую очередь карточек — курс
  «кормит» систему повторения, ничего отдельно учить не надо.

## 4. Программы курсов (самое важное)

Формат везде: урок = 400–800 слов (3–7 минут с телефона), у каждого урока квиз
3–5 вопросов с объяснениями, у большинства — блок «попробуй сам». Все примеры
привязаны к практике пользователя. Новые термины уходят в глоссарий (новые
категории: `prompting`, `ai-core`, `eq`).

### 4.1 Курс «Промпт-инжиниринг» (~26 уроков, 7 модулей)

Опора: официальные гайды Anthropic и OpenAI по prompt engineering (перед написанием —
свежая проверка через веб-поиск), собственная практика Claude Code.

**М1. Как думает языковая модель** (фундамент, без него техники — карго-культ)
1. Предсказание следующего токена: что LLM делает на самом деле
2. Токены и контекстное окно: почему модель «забывает» и как это учитывать
3. Температура и параметры генерации (связь с существующей темой ai-services)
4. Галлюцинации: почему модель уверенно ошибается и что с этим делать

**М2. Анатомия хорошего промпта**
1. Четыре кирпича: роль, задача, контекст, формат ответа
2. Конкретность: «сделай лучше» vs измеримые требования
3. Примеры в промпте (few-shot): самый недооценённый приём
4. Структура длинных промптов: разделители, секции, порядок частей

**М3. Продвинутые техники**
1. «Думай шаг за шагом»: chain-of-thought и когда он нужен
2. Декомпозиция: цепочка промптов вместо одного гигантского
3. Структурированный вывод: JSON, таблицы, шаблоны
4. Самопроверка: просить модель критиковать свой ответ
5. Роли и персоны: что реально работает, а что миф

**М4. Итерация и отладка промптов**
1. Почему первый промпт почти никогда не идеален
2. Читаем плохой ответ: как найти дырку в промпте
3. Антипаттерны: противоречия, перегруз, скрытые ожидания
4. Тестирование промптов на наборе примеров — это же QA! (мостик к профессии)

**М5. Промпты для вайб-кодинга** (связка с темой vibe-practice)
1. Контекст проекта: CLAUDE.md — это системный промпт твоего проекта
2. Как ставить задачу агенту: маленькими шагами, с критериями готовности
3. Ревью результата: что спрашивать у AI о его же коде
4. Когда всё сломалось: промпты для восстановления

**М6. Промпты для работы QA**
1. Генерация тест-кейсов и чек-листов по требованиям
2. Баг-репорт с помощью AI: быстрее и чище
3. Анализ логов и стектрейсов
4. Граница доверия: как проверять то, что AI сгенерил для работы

**М7. Системные промпты, агенты, безопасность**
1. Системный vs пользовательский промпт: кто главнее
2. Промпт-инъекции: как «взламывают» AI-приложения (QA-взгляд!)
3. Приватность: что нельзя отправлять в модель и почему

### 4.2 Курс «Нейросети и автоматизация» (~28 уроков, 8 модулей)

Опора: классические объяснения (нейрон → веса → обучение), актуальный ландшафт
моделей на дату написания (проверить веб-поиском — ландшафт меняется быстро).

**М1. Нейросети на пальцах**
1. Нейрон и веса: как числа превращаются в «решения»
2. Обучение: как сеть учится на ошибках (градиент — без математики, на аналогиях)
3. Почему это «чёрный ящик» и почему никто не знает точно, как модель решила

**М2. Как устроены LLM**
1. От нейросети к языковой модели: обучение на текстах интернета
2. Дообучение и RLHF: почему модель вежливая и слушается инструкций
3. Эмбеддинги: как смысл слов превращается в числа
4. Мультимодальность: как модель «видит» картинки

**М3. Карта мира AI**
1. Семейства моделей: GPT, Claude, Gemini, Llama — закрытые vs открытые
2. Как выбирать модель: цена / скорость / качество, маленькие vs большие
3. Reasoning-модели: что значит «модель думает перед ответом»
4. Где AI живёт: чат, API, IDE-агенты, локальный запуск

**М4. Ограничения и риски**
1. Отсечка знаний и поиск в интернете
2. Контекст не резиновый: пределы и деградация на длинном контексте
3. Недетерминированность: почему два одинаковых вопроса → разные ответы
4. Этика коротко: авторские права, приватность, дипфейки — что важно знать

**М5. AI + твои данные: RAG**
1. Почему модель не знает твои файлы и твою компанию
2. Векторный поиск на пальцах: «найди похожее по смыслу»
3. RAG vs длинный контекст vs дообучение: что когда выбирать

**М6. Агенты и инструменты**
1. Function calling: как модель «нажимает кнопки»
2. Агентный цикл: план → действие → проверка результата → повтор
3. MCP: универсальный разъём между AI и сервисами
4. Claude Code изнутри: разбираем агента, которым ты пользуешься каждый день

**М7. Автоматизация: заставляем AI работать**
1. Что стоит автоматизировать: критерии (повторяемость, объём, цена ошибки)
2. No-code: n8n / Zapier / Make — конструкторы автоматизаций
3. Свой первый сценарий через API (Gemini из Google AI Studio — у пользователя
   уже есть ключ, связка с темой ai-services)
4. Автоматизация с Claude Code: скрипты, hooks, запланированные задачи

**М8. AI в работе QA**
1. Генерация тестовых данных и сценариев
2. AI-помощь в регрессии и анализе требований
3. Где AI в QA ошибается: границы применимости (критическое мышление)

### 4.3 Курс «Эмоциональный интеллект» (~27 уроков, 8 модулей)

Опора — только проверенные рамки, без эзотерики: модель Майера-Саловея и Гоулмана
(4 навыка), регуляция эмоций по Дж. Гроссу (когнитивная переоценка), ненасильственное
общение Розенберга (ННО), исследования по активному слушанию и обратной связи.
Тон — практический: каждый урок заканчивается упражнением «попробуй сам».

**М1. Что такое EQ и зачем он тестировщику**
1. EQ = 4 навыка: замечать своё, управлять своим, замечать чужое, управлять контактом
2. Мифы: EQ ≠ «быть милым», EQ не врождённый и прокачивается
3. Почему в IT без EQ никак: баг-репорты, споры, дедлайны, удалёнка

**М2. Эмоции: матчасть**
1. Зачем эволюция дала нам эмоции: сигнальная система, а не помеха
2. Базовые эмоции и их «сообщения» (злость → нарушена граница, страх → угроза...)
3. Эмоция ≠ действие: зазор между стимулом и реакцией — главное место свободы
4. Эмоции в теле: где живёт злость и почему сначала сжимаются кулаки

**М3. Самосознание**
1. Называние эмоций (labeling): «назвал — приручил», научная база
2. Словарь эмоций: различать раздражение / обиду / разочарование / тревогу
3. Чек-ин: привычка спрашивать себя «что я сейчас чувствую?» (2 минуты в день)
4. Триггеры: составляем личную карту кнопок

**М4. Саморегуляция**
1. Пауза: физиология эмоциональной волны и почему «досчитай до 10» реально работает
2. Дыхание и тело: самые быстрые ручки регуляции (физиологический вздох, 4-7-8)
3. Когнитивная переоценка: та же ситуация — другая рамка (Гросс)
4. Руминация: как выйти из мысленной жвачки
5. Стресс и выгорание: ранние признаки, разница усталость/выгорание, что делать

**М5. Эмпатия**
1. Когнитивная vs эмоциональная эмпатия: понять ≠ утонуть в чужом
2. Активное слушание: слушать чтобы понять, а не чтобы ответить
3. Невербалика: что говорит тело собеседника (и твоё на созвоне)
4. Ошибка атрибуции: «он козёл» vs «у него горит релиз» — почему мы судим людей
   строже, чем обстоятельства

**М6. Коммуникация в сложных ситуациях**
1. Я-высказывания: «ты опять сломал» → «я застрял, потому что сборка падает»
2. ННО за 15 минут: наблюдение → чувство → потребность → просьба
3. Обратная связь: как давать (конкретно, про действие, вовремя) и как принимать
4. Конфликт: де-эскалация, поиск интереса за позицией
5. Границы и «нет»: отказывать без войны и вины

**М7. EQ на работе тестировщика** (уникальный модуль под профиль владельца)
1. Баг-репорт как акт коммуникации: критикуем код, не человека
2. Спор о severity: как отстаивать баг без «войны» с разработчиком
3. Эмоции в тексте: мессенджеры, ревью, «токсичные» формулировки и их замена
4. Когда всё падает перед релизом: сохранять голову в аврале

**М8. Практика и привычки**
1. Микропрактики: 5 упражнений по 2 минуты, встроенных в обычный день
2. Дневник эмоций: простой формат, который не забросишь
3. План на 30 дней: собираем личную программу из пройденного

## 5. Как обеспечиваем качество контента (процесс)

1. **Ресёрч перед каждым курсом**: веб-поиск + чтение первоисточников
   (Anthropic/OpenAI docs для промптов; актуальный ландшафт моделей для курса AI;
   Гоулман/Гросс/Розенберг для EQ). Конспект → скелет модулей → только потом текст.
2. **Стиль как в глоссарии**: на «ты», просто, без канцелярита, каждый термин
   объяснён на месте. Аналогии из жизни, примеры из Recall / CodeCompass / QA.
3. **Квизы пишутся по тексту урока**: каждый вопрос проверяет то, что реально
   объяснялось; у каждого ответа — объяснение (учимся и на ошибке, и на успехе).
4. **Перекрёстные ссылки**: уроки ссылаются на термины глоссария; новые термины
   из курсов пополняют глоссарий и карточки.
5. **Самопроверка после каждого модуля**: перечитать как редактор (точность фактов,
   нет ли повторов, работают ли примеры), `npm run build`, прогон в браузере.
6. **Приёмка на телефоне**: после деплоя каждого курса — проверка на S25
   (читабельность, размер шрифта, квизы пальцем).

## 6. Улучшения приложения (помимо курсов)

Приоритет 1 (обязательно):
- **Экспорт/импорт прогресса** — кнопки в настройках: скачать JSON со всем
  localStorage-состоянием / загрузить обратно. Критично: страховка от потери
  прогресса и мост при переходе PWA → APK. Делается ДО всего остального.
- **Стрик** 🔥 — дни подряд с хотя бы одним повторением или уроком; на главной.
  (Как в Recall — пользователю знакомо и это работает.)
- **Блок «Продолжить обучение»** на главной — вход в текущий урок одной кнопкой.

Приоритет 2 (желательно):
- Поиск в глоссарии также по `detail` (сейчас только term + short).
- Экран «Настройки/О приложении»: экспорт/импорт, сколько терминов/уроков, версия.
- Статистика повторений: сколько повторено сегодня/за неделю, сколько карточек
  в каждой стадии (new/learning/review).
- Плавная анимация раскрытия карточки (полировка UX).

Не делаем (осознанно): аккаунты, облачную синхронизацию, Supabase — личный
инструмент, localStorage + экспорт достаточно.

## 7. APK для Samsung S25

Два пути, рекомендация — вариант A:

**Вариант A (рекомендую): TWA через PWABuilder** — «обёртка» вокруг уже живого
сайта на Vercel.
- Как: pwabuilder.com → вставить https://codecompass-ruddy.vercel.app → Package
  for Android → скачать APK + ключ подписи; файл `assetlinks.json` положить в
  `public/.well-known/` и задеплоить (убирает адресную строку Chrome).
- Плюсы: не нужен Android SDK на компьютере; APK маленький; данные — тот же
  Chrome-origin, т.е. прогресс из установленной PWA сохраняется; обновления
  приложения = обычный деплой на Vercel, APK переустанавливать не надо.
- Минусы: первый запуск требует интернет; офлайн — через сервис-воркер PWA
  (уже настроен vite-plugin-pwa).
- Установка: скачать APK на телефон → разрешить установку из файла → готово.
  В Google Play не публикуем (личный инструмент).

**Вариант B (запасной): Capacitor** — веб-код упакован ВНУТРЬ APK, полный офлайн.
- Минусы: нужен JDK + Android SDK + Gradle на этом ПК; хранилище WebView отдельное
  от Chrome → прогресс переносить экспортом/импортом; каждое обновление контента =
  пересборка и переустановка APK. Для приложения, которое обновляется контентом
  каждую сессию, это неудобно — поэтому вариант запасной.

⚠️ В любом случае сначала — экспорт/импорт прогресса (Фаза 0). Ключ подписи APK
хранить вне git (папку с ключами добавить в `.gitignore`).

## 8. Порядок работ (фазы)

Каждая фаза заканчивается: `npm run build` зелёный → показ пользователю → коммит и
деплой только с его согласия. Одна фаза ≈ одна-две сессии.

- ✅ **Фаза 0 — страховка данных**: экспорт/импорт прогресса + экран настроек
  (`src/lib/backup.ts`, `src/features/settings/`, шестерёнка на главной).
- ✅ **Фаза 1 — движок курсов**: типы (`Course/Lesson/QuizQuestion` в types),
  прогресс (`src/lib/courseProgress.ts`), вкладка «Курсы», экраны курса и урока,
  квиз с объяснениями. Пилот: модуль М1 «Как думает языковая модель» (4 урока,
  16 вопросов, 5 новых терминов в категории `prompting`). Проверено вживую
  Playwright-прогоном (21 шаг, включая пробы) — рецепт в `.claude/skills/verify/`.
  Ожидается обратная связь пользователя по подаче уроков.
- ✅ **Фаза 2 — курс «Промпт-инжиниринг» целиком** (готов). Обратная связь
  по пилоту: «больше информации» → формат уплотнён. Ресёрч по гайдам
  Anthropic/OpenAI (июль 2026); М1 углублён (уроки 6–7 мин вместо 4–5);
  написаны М2 «Анатомия» (4), М3 «Техники» (5), М4 «Отладка» (4),
  М5 «Вайб-кодинг» (4), М6 «QA» (4), М7 «Агенты и безопасность» (3).
  Итог: 7 модулей / 28 уроков / 112 квиз-вопросов, +8 терминов
  (242 всего). Уроки М5/М6 ссылаются на термины vibe-practice и qa-tools.
  Проверено живым прогоном (структура, квизы, чипы терминов, карточки).
- 🔶 **Фаза 3 — курс «Нейросети и автоматизация»** (в работе, половина готова).
  Ресёрч ландшафта июля 2026 выполнен (флагманы: GPT-5.5, Claude Opus 4.8 /
  семейство Claude 5, Gemini 3.1 Pro, Grok 4.3; открытые: DeepSeek, GLM, Qwen;
  локально: Ollama / LM Studio). Написаны М1 «Нейросети на пальцах» (3 урока),
  М2 «Как устроены LLM» (4), М3 «Карта мира AI» (4), М4 «Ограничения и риски»
  (4) — итого 15 уроков; новая категория глоссария `ai-core` (+12 терминов,
  254 всего). Осталось: М5 «RAG», М6 «Агенты и инструменты», М7 «Автоматизация»,
  М8 «AI в работе QA».
- ⬜ **Фаза 4 — курс «Эмоциональный интеллект»**.
- ⬜ **Фаза 5 — мотивация и полировка**: стрик, «Продолжить обучение», статистика,
  поиск по detail, мелкий UX.
- ⬜ **Фаза 6 — APK**: assetlinks.json → PWABuilder → установка на S25 →
  инструкция в ROADMAP.

Почему контент по одному курсу за фазу: каждый курс — это ~25 уроков хорошего
текста + ~100 квиз-вопросов + ~30 новых терминов. Делать все три разом = потерять
качество. Каждая фаза даёт готовый, полезный кусок на телефоне.

## 9. Риски и открытые вопросы

- **Потеря localStorage** — закрывается Фазой 0 (и привычкой изредка жать «экспорт»).
- **Устаревание AI-контента** (модели меняются каждые полгода) — в уроках про
  конкретные модели давать «снимок на дату» + учить принципам выбора, а не именам.
- **EQ — чувствительная тема** — держаться исследований, не обещать «прокачку за
  неделю», не лезть в терапию (урок про выгорание честно говорит, когда пора к
  специалисту, а не к приложению).
- **Вопрос пользователю (не блокирует старт)**: подтвердить вариант APK
  (A — обёртка вокруг сайта, рекомендую / B — полностью автономный APK).
