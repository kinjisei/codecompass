# Как получить APK для телефона (CodeCompass)

> Коротко: CodeCompass — это PWA. Проще всего он уже ставится как приложение
> прямо из браузера («Добавить на главный экран»). Если нужен именно
> **APK-файл** (настоящее Android-приложение, без адресной строки Chrome) —
> собираем его через **PWABuilder** из живого сайта. Собрать APK локально на
> этом компьютере нельзя: нет Android SDK, нет `adb`, а Java только 8-й версии
> (для сборки нужен JDK 17+). PWABuilder делает сборку в облаке — ничего
> ставить не надо.

## Вариант 0 (самый быстрый): установить PWA без APK

На телефоне уже стоит CodeCompass как PWA. Если нет:

1. Открой на телефоне **https://codecompass-ruddy.vercel.app** в Chrome.
2. Меню (⋮) → **«Установить приложение»** / «Добавить на главный экран».
3. Готово — иконка на рабочем столе, запускается без адресной строки,
   работает офлайн. Обновления прилетают сами при каждом деплое.

Этого достаточно для личного использования. APK нужен, только если хочется
именно файл-установщик (например, поделиться или поставить как обычное
приложение).

## Вариант 1 (рекомендую): APK через PWABuilder

Собирает TWA-обёртку вокруг живого сайта. Прогресс сохраняется — это тот же
origin, что и установленная PWA.

### Шаг 1. Сгенерировать пакет

1. Открой на компьютере **https://www.pwabuilder.com**.
2. Вставь адрес: `https://codecompass-ruddy.vercel.app` → **Start**.
3. PWABuilder проверит манифест (у нас он уже готов: иконки, имя, цвета,
   `id`, `scope`, `orientation`) → вкладка **Android** → **Generate Package**.
4. В диалоге Android оставь настройки по умолчанию (Package ID вида
   `app.vercel.codecompass_ruddy.twa` — можно оставить как есть) →
   **Download**.

Скачается ZIP. Внутри:
- **`app-release-signed.apk`** (или `.aab`) — сам установщик;
- **ключ подписи** (`signing.keystore` + пароли в `signing-key-info.txt`) —
  ⚠️ сохрани в надёжном месте ВНЕ git: он нужен, чтобы в будущем выпускать
  обновления APK той же подписью;
- **`assetlinks.json`** — файл для связывания сайта и приложения.

### Шаг 2. Прописать assetlinks.json (убирает адресную строку)

В репозитории уже есть заготовка `public/.well-known/assetlinks.json`
(сейчас пустой `[]`). Замени её содержимое на то, что дал PWABuilder
(там будет отпечаток `sha256_cert_fingerprints` твоего ключа), закоммить и
задеплой (обычный `git push`). После деплоя Android увидит связь и запустит
APK **без строки браузера сверху**.

Проверить, что файл доступен, можно так (после деплоя):
`https://codecompass-ruddy.vercel.app/.well-known/assetlinks.json` — должен
отдавать твой JSON, а не пустой массив.

### Шаг 3. Установить APK на телефон

1. Перекинь `app-release-signed.apk` на телефон (кабель, Telegram себе,
   Google Drive — как удобно).
2. Открой файл на телефоне → разреши «установку из этого источника» →
   **Установить**.
3. Готово: CodeCompass как обычное Android-приложение.

Обновления контента при этом всё равно прилетают сами (APK — обёртка вокруг
сайта, пересобирать APK на каждый деплой НЕ нужно). APK переустанавливают,
только если поменялись иконки/имя/сам PWABuilder-пакет.

## Вариант 2 (запасной): полностью автономный APK (Capacitor)

Упаковывает веб-код ВНУТРЬ APK — работает без интернета с первого запуска.
Минусы существенные: нужен JDK 17+ и Android SDK на компьютере (сейчас их
нет), хранилище WebView отдельное от Chrome (прогресс переносится через
экспорт/импорт из «Настроек»), и каждое обновление контента = пересборка и
переустановка APK. Для приложения, которое часто пополняется, это неудобно —
поэтому вариант запасной. Если понадобится — поставим JDK 17 + Android
command-line tools и соберём через Capacitor.

## Резюме

- Нужно «просто на телефоне» → **Вариант 0** (PWA, уже работает).
- Нужен APK-файл → **Вариант 1** (PWABuilder, 10 минут, я подготовил манифест
  и `.well-known`).
- Нужен офлайн-APK без сайта → **Вариант 2** (Capacitor, требует установки
  Android-тулчейна).
